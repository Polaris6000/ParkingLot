.jsp 파일명 관련 
1. 리눅스 환경에서 대소문자 구분 엄격 // 예) 파일명이 UserLogin.jsp인데 주소창에 userlogin이라고 치면 404 에러 나요.
2. 구글 검색 엔진(SEO): 구글은 하이픈(-)을 단어 구분자로 확실하게 인식하는 반면 언더바(_)는 단어를 하나로 이어주는 것으로 처리하는 경우가 많아서 검색 노출에 미세하게 불리할 수 있다는 이유에서 
3. 실무 환경에서는  언더바(_) 대신 하이픈(-) 쓴대요. 이게 이름이 케밥 케이스(kebab-case, 하이픈 사용)래요. 예) example-file-name.html 처럼 쓰는 방식
4. 고로 저희 .jsp 파일명은 케밥 케이스로 작업하는건 어떨지 건의 드립니다.
